    CSS世界：流的破环与保护

    发表于 2022-09-29 更新于 2022-10-19 分类于 读书笔记 本文字数： 6.3k 阅读时长 ≈ 6 分钟
    对《CSS世界》第六章知识点的简要总结

    1. float

    1.1 float本质与特性

    • CSS2 属性的设计都是为图文展示服务，浮动的本质就是为了实现文字环绕效果

    • 特性：

      1. 包裹性；

      2. 块状化并格式化上下文：，元素float 的属性值不为none，则其display 计算值就是block或者table；

      3. 破坏文档流；

      4. 没有任何margin 合并；

    1.2 作用机制

    要使得内容可以和浮动元素在一个水平线上：

    • “高度塌陷”

    • 行框盒子和浮动元素的不可重叠性

    1.3 深入的作用机制

    • 浮动锚点：float 元素所在的“流”中的一个点，这个点本身并不浮动，就表现而言更像一个没有margin、border 和padding 的空的内联元素。

    • 浮动参考：浮动元素对齐参考的实体。float 元素的“浮动参考”是“行框盒子”，也就是float 元素在当前“行框盒子”内定位。

    2. clear

    2.1 clear属性

    • none：默认值，左右浮动来就来。

    • left：左侧抗浮动。

    • right：右侧抗浮动。

    • both：两侧抗浮动。

    clear 属性让自身不能和前面的浮动元素相邻，凡是可以用left和right的地方，都能用both代替

    2.2 clear缺陷

    • 如果clear:both 元素前面的元素就是float 元素，则margin-top 负值即使设成-9999px，也不见任何效果。

    • clear:both 后面的元素依旧可能会发生文字环绕的现象。

    3. BFC

    3.1 BFC定义

    • BFC全称为block formatting context，即块级格式化上下文。表现规则：BFC 特性的元素的子元素不会受外部元素影响，也不会影响外部元素。

    • 常见的触发BFC的情况：

      • html根元素；

      • float 的值不为none；

      • overflow 的值为auto、scroll 或hidden；

      • display 的值为table-cell、table-caption 和inline-block 中的任何一个；

      • position 的值不为relative 和static。

    3.2 BFC与流体布局

    和基于纯流体特性实现的两栏或多栏自适应布局相比，基于BFC 特性的自适应布局有如下优点：

    1. 自适应内容由于封闭而更健壮，容错性更强

    2. 自适应内容自动填满浮动以外区域，无须关心浮动元素宽度，可以整站大规模应用

    能兼顾流体特性和BFC 特性来实现自适应布局的属性：

    1. float:left。浮动元素本身BFC 化，然而浮动元素有破坏性和包裹性，失去了元素本身的流体自适应性，因此，无法用来实现自动填满容器的自适应布局。

    2. position:absolute。难以使用

    3. overflow:hidden。常用，但如果子元素要定位到父元素的外面可能会被隐藏

    4. display:inline-block。会让元素尺寸包裹收缩，不是我们想要的block 水平的流动特性。

    5. display:table-cell。需要IE8 及以上版本的浏览器；对连续英文字符不友好，解决：

    6. display:table-row。对width 无感，无法自适应剩余容器空间。

    7. display:table-caption。此属性一无是处。

    总结：有两套IE7 及以上版本浏览器适配的自适应解决方案：

    1. .lbf-content { overflow: hidden; }

    2. ```css
      .lbf-content {
      display: table-cell; width: 9999px;
      /* 如果不需要兼容IE7，下面样式可以省略 */
      *display: inline-block; *width: auto;
      }

    • -webkit-私有前缀支持良好的浏览器可以实现多行文字打点效果

    5. position:absolute

    absolute和float都具有“包裹性”、“块状化”、“破坏性”等特性。

    5.1 absolute的包含块

    包含块：元素用来计算和定位的一个框。

    • 元素包含块的规则：
      1. 根元素被称为“初始包含块”，其尺寸等同于浏览器可视窗口的大小。
      2. 对于其他元素，如果position是reactive或static，则包含块由其最近的块容器祖先盒的content box边界组成。
      3. 如果元素position为fixed，则包含块是初始包含块。
      4. 如果元素position为absolute，则包含块由最近不为static的祖先元素建立：如果该祖先元素是纯inline元素，假设给内联元素的前后各生成一个宽度为0 的内联盒子，则这两个内联盒子的padding box外面的包围盒就是内联元素的包含块、如果该内联元素被跨行分割了，那么包含块是未定义的；否则，包含块由该祖先的padding box 边界形成。
    • 和常规元素相比，absolute绝对定位元素包含块的差异：
      1. 内联元素可以作为包含块所在的元素。
      2. 包含块所在的元素不是父级块元素。
      3. 边界是padding box而不是content box。

    5.2 具有相对特性的无依赖absolute绝对定位

    • absolute 是非常独立的CSS 属性值，其样式和行为表现不依赖其他任何CSS 属性就可以完成。
    • 一个绝对定位元素，没有任何left/top/right/bottom 属性设置，并且其祖先元素全部都是非定位元素，其位置仍在当前位置。
    • 用途：
      1. 各类图标定位。借助margin负值。例子
      2. 超越常规布局的排版。例子
      3. 下拉列表定位。例子
      4. 占位符效果模拟。IE9 及其以下浏览器不支持placeholder 占位符效果，进行模拟
      5. 比较好的做法是使用label标签和输入框关联并覆盖在输入框上面，好处是点击占位文字输入框天然focus，并且不会污染输入框的value。
    • 浮动与绝对定位结合使用有时浏览器表现不一致，属于“未定义行为”。

    6. absolute与overflow

    1. 如果overflow不是定位元素，同时绝对定位元素和overflow容器之间也没有定位元素，则overflow无法对absolute元素进行剪裁。
    2. 如果overflow的属性值不是hidden而是auto或者scroll，即使绝对定位元素高宽比overflow元素高宽还要大，也都不会出现滚动条。
    3. 由于position:fixed固定定位元素的包含块是根元素，因此，除非是窗体滚动，否则上面讨论的所有overflow 剪裁规则对固定定位都不适用。
    4. transform除了改变overflow属性原有规则，对层叠上下文以及position:fixed的渲染都有影响。

    7. absolute与clip

    clip属性要想起作用，元素必须是绝对定位或者固定定位，也就是position 属性值必须是absolute或者fixed。

    7.1 clip属性

    clip剪裁非常有用，在以下两种场景下具有不可替代的地位：

    1. fixed 固定定位的剪裁：因为fixed固定定位元素的包含块是根元素，除非是根元素滚动条，普通元素的overflow是根本无法对其进行剪裁的。

    2. 最佳可访问性隐藏：指的是虽然内容肉眼看不见，但是其他辅助设备却能够进行识别和访问的隐藏。cilp具有更强的普遍适应性，任何元素、任何场景都可以无障碍使用。

    7.2 深入了解clip的渲染

    clip隐藏仅仅是决定了哪部分是可见的，非可见部分无法响应点击事件等；然后，虽然视觉上隐藏，但是元素的尺寸依然是原本的尺寸，在IE浏览器和Firefox浏览器下抹掉了不可见区域尺寸对布局的影响，Chrome浏览器却保留了。

    8.absolute的流体特性

    • 当 absolute遇到left/top/right/bottom属性的时候，absolute元素才真正变成绝对定位元素。如果我们仅设置了一个方向的绝对定位，另一个方向依然保持相对特性。
    • 对立方向同时发生定位的时候绝对定位元素也具有与普通块级元素类似的流体特性。
    • 当绝对定位元素处于流体状态的时候，各个盒模型相关属性的解析和普通流体元素都是一模一样的。绝对定位元素的margin:auto的填充规则和普通流体元素的一模一样。

    9. position:relative

    9.1 relative对absolute的限制

    relative/absolute/fixed都能对absolute的“包裹性”以及“定位”产生限制，只有relative可以让元素依然保持在正常的文档流中。

    9.2 reactive与定位

    reactive定位两大特性：

    1. 相对自身：相对于自身位置进行偏移定位。（但百分比值相对于包含块）
    2. 无侵入：当relative进行定位偏移的时候，一般情况下不会影响周围元素的布局。

    当相对定位元素同时应用对立方向定位值的时候，只有一个方向的属性会起作用。

    9.3 relative的最小化影响原则

    1. 尽量不使用relative，如果想定位某些元素，看看能否使用“无依赖的绝对定位”。
    2. 如果场景受限，一定要使用relative，则该relative务必最小化。

    10. position:fixed固定定位

    10.1 不一样的包含块

    • position:fixed固定定位元素的“包含块”是根元素，我们可以将其近似看成html元素。
    • 利用absolute/fixed元素没有设置left/top/right/bottom的相对定位特性，可以将目标元素定位到我们想要的位置。

    10.2 position:fixed的absolute模拟

    我们希望元素既有不跟随滚动的固定定位效果，又能被定位元素限制和精准定位。原理：页面的滚动使用普通元素替代，此时滚动元素之外的其他元素就有了“固定定位”的效果。
